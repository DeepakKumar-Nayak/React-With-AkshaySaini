1. What are dependencies
--------------------------
"A dependency is an external software package or module that an application requires to function correctly. It is a piece of code written by third-party developers that we import into our project to handle specific tasks, such as state management, UI rendering, or API calls."

"Dependencies are pre-written code modules that our application leverages to perform specific functionalities. Instead of writing everything from scratch, we use these reusable packages to speed up development and ensure stability. These are tracked via a manifest file called package.json."

 npm init:
 --------
 "The npm init command is used to initialize a new or existing npm package. It is an interactive utility that guides you through the creation of a package.json file."


It sets up the basic configuration for your project, defining metadata like the project's name, entry point (e.g., index.js), and author. Without this initialization, the project cannot manage external dependencies via NPM."

2. English Mein (Technical Interview Version)
"A Transitive Dependency occurs when your project depends on Library A, 
and Library A itself depends on Library B. Even though you didn't explicitly install Library B, 
it becomes a part of your project's node_modules because Library A cannot function without it."

"NPM manages these transitive dependencies by building a Dependency Tree. It ensures 
that all the required sub-packages are fetched and stored so that the primary package (like Parcel) works as expected.

How do you check the Transitive Dependency:
--------------------------------------------
Aap 3 tareekon se check kar sakte ho:

node_modules folder: Aap dekhoge ki aapne sirf parcel dala tha, 
lekin folder mein 100 aur folders aa gaye.

package-lock.json: Is file ko khologe toh wahan har package ke niche ek 
requires ya dependencies ka section hoga. Wohi transitive dependencies hain.

Command: Terminal mein npm list chalao. Ye aapko ek poora tree dikhayega ki kaun kis par depend hai.
